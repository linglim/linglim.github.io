java基础

Date 2016-11-14(星期一) 00:00 By Linglim Category java语言. Tags java

Summary：基础

一、基础：

  1. 编译与解释型 编译：一次性生成机器码、特定平台、脱离开发环境、效率高、无法发移植 解释：执行程序前都会编译一遍程序、效率低、不能脱离解释器独立运行、跨平台易。 java:先编译成与平台无关的字节码（.class）,字节码是面向虚拟机的，字节码是一样的；java虚拟机再解释字节码，虚拟机是面向不同平台，jvm相当于一个转换器，将相同的字节码在不同平台运行。 jvm:抽象的计算机，指令集、寄存器、类文件格式、栈、垃圾回收堆、存储区

  2. jvm: jre:包含jvm+类加载器+字节码校验器+基础类：只可运行java程序 jdk:包含jre+java开发工具:java开发，有编译的命令工具

  3. 命令

  • javac：编译java程序

    javac -d destdir srcfile #destdir 指定.class存放路径。srcfile：java源文件位置

  • java ：执行程序 java java类名

4.path： path:环境变量：win操作系统根据path环境变量查找命令(javac和java) classpath：jre寻找java.class,需要加（.），编译和运行java时需要lib路径下dt.jar和tool.jar的类。需要加将这两个添加到变量里。1.4以前版本没有自动搜索的功能。1.5以后都可不用设置classpath。

5 . 程序组织

类是最小程序单位。 最简单java程序： class Test{}//可通过编译，但不可被解释运行报错。

解释器规定：必须从包含public static void main（String[] args）方法开始执行。 源文件名与public类名同名

6 . 垃圾回收机制： jre后台自动回收无用内存（无对象引用）

7.UML：统一建模语言

  • 用例图：描述系统的一个功能模块 组成：用例（椭圆）、角色、关系

  • 类图：矩形

    • 组成：类名、类属型、类方法

    • 关系：

      • 关联： 聚合（空心菱形），实体可同时是另一个实体的部分 组合（实心菱形）：实体不可同时是另一个实体的部分

      • 泛化（继承）：空心三角

      • 依赖：带箭头的虚线，指向被依赖的实体

  • 组件图：可复用的组建打包成jar、war 组成：组件、接口（圆圈）、port(小矩形)

  • 部署图：物理运行情况建模，用来部署或安装软件 组成：节点（设备：无计算能力，处理器）

  • 顺序图：垂直维度：时间+水平维度：消息被发送到对象实例 组成：顶部==实例：类名,

  • 演化图：活动图（并发+流程图，描述原理、业务逻辑以及工作流技术）+状态机图

8 . 数据类型=基本数据类型（boolean+数值类型）+引用数据类型（类、接口、数组类型） 直接量：=基本类型+字符串类型+null类型

9 .数组与内存控制：引用类型。数组变量是个引用，引用指向真实的数组内存。 数组元素和数组变量在内存分开存储，数组变量：栈内存（局部变量+），数组元素：堆内存 。 数组初始化：静态初始化：直接赋值，动态初始化：只赋值长度。系统默认赋值。基本类型数组初始化：元素值直接存储在对应的数组元素中。引用类型初始化：数组元素存储的还是引用。 所有的引用变量不需要初始化，需要进行初始化的是引用变量所引用的对象。

2、== ： 基本类型：类型和值都相同 引用类型：指向同一个对象（不可比较没有父子关系的对象） Object 类提供的equals方法：与==没有区别 String的equals方法重写：字符串值相等 jvm用常量池管理字符串直接量、常量（列“hello”）。常量池保证相同的字符串直接量就一个，无副本。

String s1="疯狂java";
String s2="疯狂"+"java";
String s3=new String("疯狂java");
s1==s2//true:编译期就确定（常量池）
s1==s3//false：new 在运行期创建（对内存）

3、类成员 java类里包含：Field、方法、构造器、初始化块、内部类（接口、枚举）5种成员。 类里static修饰的成员就是类成员、静态初始化也是。

4、单例类：只能创建一个实例，不允许自由创建该类的实例。

  • 创建：构造器用private修饰，提供public static xxx getInstance方法，private static xxx instance 变量缓存已创建的对象。

  • 使用：只能用getInstance得到实例 Singleton s1=Singleton.getInstance();

5、final修饰符:被修饰的类、方法、变量不可变 final成员变量：可在变量被定义、初始化块、构造器中指定初始值。

  • 类field:静态初始化块或声明时指定初值

  • 实例field:非静态初始化块、声明、构造器 final 局部变量：可在定义时、也可不指定默认值，但后序只能赋值一次 final基本类型：值一直不变 final引用类型:引用类型地址确定，但值会变。

可进行宏替换的final变量：直接量

  • 条件：用final修饰，定义时赋初值，初始值在编译时确定，或只用基本算数表达式、字符串连接运算、没有访问普通变量、调用方法。 final方法：不可被重写，如父类的方法不希望子类重写 final类：不可以有子类。 不可变类：创建该类的实例后，实例的Field是不可变的。 定义如下:private和final修饰field,提供带参数的构造器，只有getter方法，有必要的话要重写equals和hascode的方法。 缓存实例的不可变类;某个对象要重复使用，需要缓存。不会重新生成这个对象。

6、抽象类和抽象方法：abstract修饰。有抽象方法的只能被定义抽象类，抽象类可以没有抽象方法。 抽象类不可被实例化，构造器不能被用来创造实例，只能被子类调用，子类实现方法，不能用static和abstract不可同时出现. 抽象类：可以有普通方法，静态方法、构造器、初始化等

7、接口: 只能有抽象方法和常量定义。接口只能继承接口。

[修饰符]interface 接口名  extends 父接口1，父接口2，...
{0-n个常量定义；
 0-n个抽象方法定义；
内部类（内部接口、枚举）
}
成员可省略修饰符、默认public，只能是public
field 由不写默认，默认public static final定义，附初值

使用：不能用于创建实例，单可以用于声明引用类型变量

实现： 【修饰符】 class 类名 extends 父类 implements 接口1…n {类体：完全实现接口的抽象方法，否则保留从父类里继承的抽象方法，此类设为抽象类} public 修饰

8、内部类：一个类放到另一个类里，适用于创建仅需要一次使用的类。 内部类可以访问外部类的私有数据，外部类访问则需显示创建对象， 静态内部类：属于外部类本身，不属于某个对象，可使用外部类成员。外部类调用类名访问内部类的成员。非静态内部类属于外部类的实例。

使用：

  • 外部类内使用内部类：new 创建实例

  • 外部类以外使用非静态内部类： 定义：Outerclass.innerclass varNAME Outerinstance.new InnerConstructor()

  • 外部类以外使用静态内部类： new OuterClass.INnerConstructor()

局部内部类：放在方法里定义，不适用访问控制符修饰和static.只能在外 部类方法内使用

匿名内部类：值需要一次使用的类 new 父类构造器（实参列表）或实现接口（）{ //匿名内部类实体部分}

闭包和回掉：

枚举类：实例有限且固定的类。不能派生子类 使用enum定义，默认使用final修饰，不能派生子类，构造器只能用private访问控制符，所有实例第一行显示列出，自动public static final 修饰。
